# 构图模式与原创规则

## 先选结构，不先选花样

生成前先判断逻辑关系，再选一种主结构：

- 层级包含：树状 / 分层 / 上下堆叠
- 交集关系：双圆 / 重叠区域 / 中间共享区
- 二维分类：四象限 / 二维格
- 主题下并列分类：中心分组 / 放射分组 / 云朵分组
- 先后步骤：线性流程 / 路径 / 接力结构

如果一段内容同时像流程、层级、交集三种东西，说明拆解还不够干净，要先重写，再画。

一张图只用一个主结构，不要把树状图、流程图、四象限硬塞在一起。

## 基础结构类型

选择一种结构即可，不要混太多。

### Workflow 流程

适合：输入 -> 处理 -> 输出，内容生产，AI 工作流，自动化链路。

画法：左侧输入，中间mychouchou或怪机器处理，右侧输出，橙色箭头表达主流向。

### 系统局部

适合：信息来源、过滤器、数据库、渲染器、agent 系统局部。

画法：只画 3-5 个核心模块，mychouchou参与其中一个关键动作。

### 前后对比

适合：混乱/有序，手动/自动，分散/收拢，焦虑/稳定。

画法：左混乱，右稳定，中间橙色箭头。角色可以更夸张。

### 角色状态

适合：用户痛点、创作者状态、工具太多、信息焦虑、卡住到跑起来。

画法：2-4 个小状态，每个状态一个短标注。

### 概念隐喻

适合：内容工厂、信息仓库、脑内黑盒、工作流机器、自动日报。

画法：一个大的怪物件或机器，少量输入，一个输出。要有记忆点。

### 方法分层

适合：方法论框架、系统层级、能力栈、内容系统分层。

画法：一层层盒子，不要正式金字塔；mychouchou在旁边搬、扛、搭、压，让层级真的“被建起来”。

### 地图路线

适合：从想法到上线、用户路径、内容承接路径、学习路线。

画法：一条弯曲路径，少量节点，mychouchou牵线或走路。

### 小漫画分镜

适合：失败到成功、真实过程、吐槽、使用前后变化。

画法：2-4 个小场景，每格只表达一个动作。

## 分组规则

- 每个分组只承载一个小类，不要让单个组里塞太多元素。
- 一个组内建议 3-6 个元素，超出就继续拆。
- 分组外轮廓要像手绘云朵或松弛框线，不要过度几何化。
- 组内优先“图示在上、短标签在下”，保持直觉阅读顺序。
- 组与组之间必须留白，不碰撞、不叠压。

## 原创隐喻生成法

每次都从当前文章重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成一个物理动作：卡住、漏掉、变重、分拣、沉淀、发酵、开门、折叠、拆包、回流。
2. 把系统结构换成一个低科技物件：坏掉的机器、纸箱、抽屉、水管、邮筒、怪表盘、秤、井、梯子、奇怪工位。
3. 让mychouchou承担动作：不是站旁边，而是卡在机器里、拉线、守门、搬运、修补、称重、扶梯子、记录、把东西塞进某个怪装置。

### 可用物件池

- 纸箱、抽屉、旧机器、漏斗、秤、邮筒、门、井、梯子、水管、线团、闸门、转盘、黑盒、打孔器、压面机、晾衣绳、怪工位。
- 用时只选 1-2 个主物件，不要堆满。

### mychouchou 动作池

- 拉、扛、塞、捞、压、称、缝、剪、拧、守、推、接、拆、标记、回收。
- 动作要服务核心意思，不要为了怪而怪。

## 角色置入规则

- mychouchou优先出现在标题旁的导览位，或结构里的核心动作位。
- 如果只出现一次，优先放在最关键动作处，不要放角落。
- 一张图里通常 1 只最稳，最多 2-3 次；多了就容易变成插画而不是解释图。
- 动作必须呼应结构：讲筛选就分拣，讲承接就牵线，讲过滤就守门，讲层级就搭建。
- 如果mychouchou和动作不匹配，宁可换动作，不要硬塞猫。

## 文案与排版边界

- 图像模型优先负责：角色、物件、路径、分组、箭头、留白、情绪。
- 长标题、长正文、复杂说明，尽量不要让模型直接写进图里。
- 如果用户需要“正式信息图”，先产无字骨架图，再由排版工具补字。

## 反复刻规则

不要默认打开或复刻 `assets/examples/`。这些图片只用于风格校准，例如线条密度、留白、颜色克制、mychouchou气质。

除非用户明确说“照这张 / 复刻这个构图 / 用这个案例改”，否则不要直接复用这些旧构图：

- 传送带两个断点
- mychouchou在内容机器里拉判断杆
- mychouchou分拣流量/信任/转化
- mychouchou切素材鱼
- mychouchou牵承接路径
- mychouchou拉三层信息源
- 三只mychouchou分别拿喇叭/搭桥/开门
- mychouchou盖章话术工具箱
- mychouchou举牌看常见坑路径

同类主题也要换新隐喻。例如“承接路径”不一定画路线，可以画mychouchou把内容尾巴接到门把手；“一鱼多吃”不一定画鱼，可以画mychouchou把一个纸团压成几种形状。
